# AI作業ルール

このリポジトリでは、AIエージェント（Claude / Codex）がコード作業を行う可能性があります。
以下のルールを必ず守ってください。

---

# 1. 安全ルール（最重要）

以下の操作を **ユーザーの明確な許可なしに実行してはいけません**

- ファイルやディレクトリの削除
- 大規模なリネーム
- 大量のファイル移動
- 既存コードの大規模な書き換え

以下のコマンドは禁止です

- rm -rf
- git reset --hard
- git clean -fd
- 強制的な履歴書き換え

必ず

1. 変更内容を説明する
2. 提案する
3. ユーザーの許可を待つ

---

# 2. 作業フロー

AIは以下の順序で作業してください

1. リポジトリを分析する
2. 問題点を説明する
3. 変更案を提案する
4. ユーザーの許可を待つ
5. 実装する
6. 変更内容をまとめる

勝手に実装を開始してはいけません。

---

# 3. フォルダ構造ルール

このプロジェクトの主な構造

app/
Next.js のアプリケーションコード

public/
公開アセット

docs/
ドキュメント

tools/
開発用スクリプト

AIは以下を守ってください

- 既存のフォルダ構造を勝手に変更しない
- 大きな構造変更は必ず提案する

---

# 4. コーディングルール

- 新しいコードは TypeScript を使用する
- React は関数コンポーネントを使用する
- 大きすぎるファイルを作らない
- Next.js App Router の構造を守る

---

# 5. 変更ルール

AIがコード変更を行う場合

必ず以下を提示する

- 変更の目的
- 変更するファイル
- 変更内容
- 影響範囲

変更後は

変更したファイルの一覧をまとめてください。

---

# 6. AIエージェント役割分担

このプロジェクトでは Claude と Codex を役割に応じて使い分けます。

## Claude の担当

- UI・デザイン・レイアウト
- コンポーネントの見た目・スタイル
- ユーザー体験（UX）の改善提案
- コードレビュー・リファクタリング提案

## Codex の担当

- ロジック・アルゴリズムの実装
- API・バックエンド処理
- データ取得・加工
- テスト・ユーティリティ関数

## 共通ルール

- 担当外の領域を勝手に変更しない
- 担当外の変更が必要な場合はユーザーに確認する

---

# 7. 学習・運用サポートルール

ユーザーはこのサイトを**自分で運用・保守する**立場です。
そのため、AIは「作るだけ」でなく「ユーザーが理解して自分で管理できる状態」を目指して実装してください。

## 基本方針

- ユーザーが将来、自分でコードを読み・修正し・運用できることを最優先にする
- 難解すぎる実装より、シンプルで理解しやすい実装を選ぶ
- 「なぜそう作ったか」を常に説明する

## 実装前に説明すること

- **目的**：何のためにこの実装を行うか
- **使用技術**：どのライブラリ・API・概念を使うか
- **なぜこの方法か**：他の選択肢と比較した理由
- **運用への影響**：この実装がサイト運用にどう関わるか

## 実装中に説明すること

- コードの重要な部分にはコメントで意図を記載する
- 難しい概念や初見のAPIは一言で補足する

## 実装後に説明すること

- 何を実装したかの要約
- **運用時に知っておくべきこと**（更新方法・設定変更の場所・注意点）
- ユーザーが自分でメンテナンスするために必要な知識
- 関連して知っておくと役立つ技術・概念があれば紹介する

## 説明のスタイル

- 専門用語を使う場合は簡単に補足する
- 長すぎず、要点を絞って伝える
- 「なぜそうするか」「運用時にどうなるか」を意識して説明する
